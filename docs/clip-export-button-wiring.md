# Bounce-Selected-Clips → Sidecars

A **BOUNCE** button in the StemForge device's right column. Click it from
within Live to read every non-empty clip slot in your A/B/C/D source tracks,
slice each clip from its source audio, and write per-clip
`.manifest_<hash>.json` sidecars + a `.manifest.json` BatchManifest.
Output goes to `~/stemforge/exports/<timestamp>/`.

The output is the exact format `ppak-load-from-manifest` consumes — round-trip
into the EP-133 in one command.

## What's automated (no Max-editor work)

Everything below is generated by `v0/src/maxpat-builder/builder.py` from
[v0/interfaces/device.yaml](../v0/interfaces/device.yaml) — a clean
`build_amxd → build-pkg → install` ships with the button working:

| Piece | File |
|---|---|
| BOUNCE button paint + click dispatch in the v8ui | [sf_ui.js](../v0/src/m4l-package/StemForge/javascript/sf_ui.js) — `bounceBtnRect` |
| JS module that reads clips, builds spec, spawns helper | [sf_clip_export.js](../v0/src/m4l-package/StemForge/javascript/sf_clip_export.js) |
| Python helper that slices audio + writes sidecars | [tools/m4l_export_clips.py](../tools/m4l_export_clips.py) |
| `[js sf_clip_export]` box in the patcher | builder.py — `OBJ_SF_CLIP_EXPORT` |

| `sf_clip_export` outlet 1 → `[shell]` | builder.py — connects to existing `OBJ_SHELL` |
| 6 NDJSON `export_*` routes → JS handlers | builder.py — outlets 7–12 of `OBJ_ROUTE_NDJSON` |

## How to use

In Ableton Live, with the StemForge device on a track:

1. Set up your source tracks. By default the JS looks for tracks named:
   - `SF | A` → group A
   - `SF | B` → group B
   - `SF | C` → group C
   - `SF | D` → group D
2. Drop clips into any clip slots in those tracks. Slot 1 → pad `.`, slot 2
   → pad `0`, etc. (bottom-up EP-133 layout).
3. Click **BOUNCE** in the device's right column.
4. The output goes to `~/stemforge/exports/<timestamp>/`:
   - `A00.wav`, `A01.wav`, …, `D11.wav` (one per non-empty slot)
   - `.manifest_<hash>.json` per WAV (sidecar)
   - `.manifest.json` at the root (BatchManifest)
5. Round-trip to EP-133:
   ```bash
   ppak-load-from-manifest ~/stemforge/exports/<TIMESTAMP>/.manifest.json \
     --groups A=A B=B C=C D=D --start-slot 300
   ```

## Driving from a Max message box (alternative trigger)

If you want to trigger without going through the v8ui (e.g. for debugging or
custom track names):

```
exportClips                              ← uses defaults: SF | A, SF | B, SF | C, SF | D
exportClips "My Track A","My Track B"    ← explicit names
exportClipsByIndex 2 3 5 6               ← zero-based track indices
```

Wire any message box to the `[js sf_clip_export]` box's first inlet.

## Producer-side decisions (so the consumer is dumb)

The Python helper makes all the routing decisions before writing the manifest:

| Field | How it's set |
|---|---|
| `suggested_group` | One per source track (A → first track, etc.) |
| `suggested_pad` | Slot index → bottom-up `BAR_INDEX_TO_LABEL` (`.`, `0`, `ENTER`, `1`..`9`) |
| `playmode` | `oneshot` if `bars < 0.5`, else `key` |
| `bpm` | The clip's warp BPM if warped, else project tempo |
| `time_mode` | `bpm` for `key` clips, omitted for one-shots |
| `bars` | `length_beats / signature_numerator` |
| `name` | 16-char-trimmed clip name (or filename stem) |

The consumer (`ppak-load-from-manifest`) honors `suggested_pad` /
`suggested_group` directly per the contract in
[specs/manifest-spec.md](../specs/manifest-spec.md).

## V1 limitations

- **No warp baking.** Warped clips bounce from the SOURCE audio. The clip's
  `loop_start`/`loop_end` are honored (we trim to the active loop region),
  but warp markers don't apply. True freeze-and-export is V2 — needs
  `track.freeze()` + polling + reading from `<set>/Samples/Processed/Freeze/`.
- **No effects baking.** Same as above — only the source sample is read,
  device chain output is ignored.
- **Hardcoded helper path.** `sf_clip_export.js` has
  `HELPER_PATH = "/Users/zak/zacharysbrown/stemforge/tools/m4l_export_clips.py"`
  baked in. Send `setHelperPath /alt/path.py` to override at runtime.
- **System python3.** The helper uses `/usr/bin/env python3` and imports
  `stemforge.manifest_schema` + `soundfile`. Make sure the system python
  has both installed, or send `setPythonBin "uv run --directory ..."`.
