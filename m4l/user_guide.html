StemForge — User Guide

Complete guide to splitting stems, loading them into Ableton Live, and working with pipelines.
Covers the CLI, the M4L Loader device, and day-to-day workflow.

Loader Build Guide Template Builder Guide Dev Log Setup Guide

1 Prerequisites

  • Ableton Live 12+ — the Loader uses create_audio_clip() which requires Live 12
  • Max for Live — included with Live Suite, or available as an add-on for Standard
  • Python 3.11+ via pyenv — required for the CLI
  • uv — fast Python package manager (curl -LsSf https://astral.sh/uv/install.sh | sh)
  • ffmpeg — for audio format conversion (brew install ffmpeg)
M2/ARM Macs: Make sure your terminal is running natively (not under Rosetta). Check with uname -m — it should show arm64, not x86_64. If you see x86_64, open Terminal.app's "Get Info" and uncheck "Open using Rosetta".

2 Installation

Clone and install

$ git clone https://github.com/zacharysbrown/stemforge.git $ cd stemforge $ uv venv $ source .venv/bin/activate $ uv pip install -e ".[local]"
If llvmlite fails to build: Install the pre-built wheels first, then retry:
$ uv pip install llvmlite numba librosa --no-build $ uv pip install -e ".[local]"

Verify the install

$ stemforge --help Usage: stemforge [OPTIONS] COMMAND [ARGS]... StemForge -- stem splitting + beat slicing for Ableton Live. Commands: split Split an audio file into stems and slice at beat boundaries. clean-beats Delete silent beat slices from processed folders. list Show available stems, presets, and models. balance Show remaining LALAL.AI API minutes. create-templates Build the 7 StemForge template tracks in Ableton Live. generate-pipeline-json Convert pipelines/*.yaml to *.json for M4L device.

Directory structure

StemForge creates these directories automatically:

  • ~/stemforge/inbox/ — drop raw audio files here (optional convention)
  • ~/stemforge/processed/ — output directory for all stems and manifests
  • ~/stemforge/logs/ — processing logs

4 Installing the Loader Device

  1. In your StemForge Templates set, create a new MIDI track named "SF Loader"
  2. Drag StemForgeLoader.amxd from m4l/ onto the track
  3. The device should auto-initialize with status messages in its display

Alternatively, build the device from scratch: see the Loader Build Guide.

Device controls

Load Latest

Button. Click to load the most recently processed stems from ~/stemforge/processed/. Finds the stems.json with the newest processed_at timestamp.

Watch Mode

Toggle. When enabled, the Loader polls ~/stemforge/processed/ every 3 seconds. When a new stems.json appears, it loads automatically. Toggle off to stop watching.

Text Input

Type a snake_case track name (e.g., hey_mami) and press Enter to load that specific track's stems. The name must match the folder name in ~/stemforge/processed/.

Status Display

Shows loading progress, errors, and which stems were loaded onto which tracks. Also appears in the Max Console (Window → Max Console).

5 Processing a Track

Basic usage

$ stemforge split track.wav

This separates the audio into stems, detects BPM, slices at beat boundaries, and writes a stems.json manifest. Output goes to ~/stemforge/processed/<snake_case_name>/.

Options

# Choose backend explicitly $ stemforge split track.wav --backend demucs $ stemforge split track.wav --backend lalal # Use a specific Demucs model (4-stem, fine, or 6-stem) $ stemforge split track.wav --model 6stem # Use a specific pipeline config $ stemforge split track.wav --pipeline glitch $ stemforge split track.wav --pipeline idm_crushed # Skip beat slicing (full stems only) $ stemforge split track.wav --no-slice # Skip peak normalization $ stemforge split track.wav --no-normalize # More aggressive silence threshold for beat slices $ stemforge split track.wav --silence-threshold 0.01

Supported formats

  • .wav, .mp3, .flac, .ogg — supported directly
  • .m4a, .aac — convert to WAV first:
$ ffmpeg -i track.m4a track.wav $ stemforge split track.wav

Output structure

~/stemforge/processed/hey_mami/ stems.json # manifest (BPM, paths, pipeline, metadata) drums.wav # full drum stem bass.wav # full bass stem vocals.wav # full vocal stem other.wav # full "other" stem (textures, synths, etc.) drums_beats/ # beat-sliced drum files drums_beat_001.wav drums_beat_002.wav ... bass_beats/ vocals_beats/ other_beats/

6 Loading Stems in Ableton

Method 1: Type the track name

  1. In the Loader device, click the text input field
  2. Type the snake_case name (e.g., hey_mami) and press Enter
  3. The Loader reads ~/stemforge/processed/hey_mami/stems.json and loads all stems

Method 2: Load Latest

  1. Click the Load Latest button
  2. The Loader finds the most recently processed track and loads it

Method 3: Watch Mode

  1. Toggle Watch on
  2. Run stemforge split from the CLI
  3. The Loader auto-detects the new stems.json within 3 seconds and loads automatically
  4. Toggle off when not needed to save CPU

What happens when stems load

  • BPM: session tempo is set automatically from the manifest
  • Track matching: each stem is matched to a template track by name (e.g., "drums" stem → "SF | Drums Raw" track)
  • Audio loading: stems are loaded into the first empty clip slot on each matching track via create_audio_clip()
  • Clip properties: clips are named (song_name-stem_part), warping is enabled, warp mode is set from the pipeline, and looping is turned on
  • Effect parameters: if the pipeline config includes effect settings, they are applied to the devices on each track
Loading multiple songs: Each song's stems go into the next empty clip slots on the template tracks. You can load many songs onto the same template set — they stack vertically in the Session View.
Stem-to-track mapping: The Loader uses flexible matching. The stem "drums" matches any track containing "drums" or "drum" in its name. The stem "other" matches tracks containing "texture" or "other". The stem "synthesizer" matches "synth" or "texture". See STEM_TRACK_MAP in stemforge_loader.js for the full mapping.

7 Pipeline Configs

Pipelines control which template track each stem goes to, the warp mode, and effect parameters. The pipeline name is baked into stems.json at processing time.

default
Clean stems, warped and looped, ready to use
drums → SF | Drums Raw (beats) · bass → SF | Bass (beats) · other → SF | Texture Verb (complex) · vocals → SF | Vocals (tones) · synth → SF | Texture Verb (complex) · guitar → SF | Texture Verb (complex) · piano → SF | Texture Verb (tones)
idm_crushed
Bitcrushed, saturated, degraded — maximum texture (Aphex Twin / Squarepusher territory)
drums → SF | Drums Crushed (beats) with LO-FI-AF + Decapitator + Compressor params · bass → SF | Bass (beats) with EQ + subtle tape · other → SF | Texture Verb (complex) with spectral + analog + tape echo + hall reverb
glitch
Granular reverse textures, spectral mangling (Four Tet / Boards of Canada)
drums → SF | Drums Raw (beats) · bass → SF | Bass (beats) · other → SF | Texture Crystallized (complex-pro) with Crystallizer + Reverb params · synth → SF | Texture Crystallized (complex-pro) with reverse granular
ambient
Expansive reverbs, slow modulation — textural/ambient IDM
drums → SF | Drums Raw (beats) with subtle analog warmth · bass → SF | Bass (tones) · other → SF | Texture Verb (complex-pro) with slow phase + long tape delay + plate reverb · synth → SF | Texture Verb (complex-pro)

Using a pipeline

$ stemforge split track.wav --pipeline glitch

Editing pipeline configs

  1. Edit pipelines/default.yaml (or create a new YAML file in pipelines/)
  2. Run stemforge generate-pipeline-json to convert YAML to JSON
  3. Restart or reload the M4L Loader device to pick up changes
$ stemforge generate-pipeline-json OK default.yaml → default.json
Pipeline YAML structure: Each pipeline has a name, description, and a stems section. Each stem entry specifies the template track name, color, warp mode, loop setting, and optional effects array with device index + parameter values. See pipelines/default.yaml for the full schema.

8 Cleaning Up Beat Slices

Beat slicing can produce silent or near-silent files. Use clean-beats to remove them.

# Preview what would be deleted (dry run) $ stemforge clean-beats --dry-run drums_beats/: would delete 3 / kept 45 bass_beats/: would delete 8 / kept 40 (dry run) 11 silent beats removed, 85 kept (threshold=0.001) # Actually delete silent beats $ stemforge clean-beats # More aggressive cleanup (higher threshold) $ stemforge clean-beats --threshold 0.01 # Clean a specific directory $ stemforge clean-beats --dir ~/stemforge/processed/hey_mami
Default threshold: 0.001 RMS. Beat slices with an RMS energy below this are considered silent. Use --threshold 0.01 for more aggressive cleanup (removes very quiet beats too).

9 Troubleshooting

"can't find file stemforge_loader.js"
The .amxd and .js files must be in the same folder. If the device is frozen, the JS is bundled inside. If unfrozen, ensure both files are in m4l/. After fixing the path, delete and re-create the [js] object in the Max editor.
"No stems.json found"
The Loader looks for stems.json files inside subdirectories of the processed dir. Check that: (1) you've run stemforge split at least once, (2) the processed dir path is correct (default: ~/stemforge/processed), and (3) the path uses the full absolute path in the setProcessedDir message box.
Stems loading on wrong tracks
The Loader matches stems to tracks by name. Check that your template track names contain the stem name (e.g., "drums", "bass", "vocals"). The matching is case-insensitive but the stem name must appear in the track name. See STEM_TRACK_MAP in the JS for the full matching logic (e.g., "other" also matches "texture").
CPU spike after loading
Normal. When stems are loaded, Ableton analyzes and warps the audio files. This causes a brief CPU spike. Wait a few seconds for analysis to complete. If it persists, check that warp mode is set correctly (Beats for drums, Complex/Complex Pro for textures).
textedit is blank / no status messages
Check the wiring in the Max editor: [js] outlet 0 (leftmost) must connect to [textedit] inlet 0. Also check the Max Console (Window → Max Console) — status messages always appear there via post() even if the textedit wiring is wrong.
"already loaded" (old version only)
This check was removed in the current version. If you see this message, you're using an older version of stemforge_loader.js. Send a reset message to the [js] object to clear the last-loaded state, or update to the latest JS file.
JS edits not taking effect
If the device is frozen, your edits to m4l/stemforge_loader.js will be ignored — the device uses the bundled copy. Unfreeze the device, copy the updated JS to the project's code/ subfolder (usually in ~/Documents/Max 9/Max for Live Devices/), or re-freeze to rebundle.
llvmlite / numba build failure
Install pre-built wheels: uv pip install llvmlite numba librosa --no-build
Demucs can't read .m4a
Convert to WAV first: ffmpeg -i track.m4a track.wav

Loader Messages

These messages can be sent to the [js stemforge_loader.js] object's inlet:

MessageDescription
bang Load the most recent stems (same as Load Latest button)
