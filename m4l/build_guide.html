StemForge M4L Build Guide

Step-by-step: build the Template Builder and Loader devices in the Max for Live editor.
No prior Max/MSP experience needed — just follow the steps.

DEVICE 1 Template Builder

One-time setup tool. Creates all 7 StemForge template tracks with devices, parameters, and colors automatically. Run once, then remove it.

1 Create the host MIDI track

  • Open your StemForge Templates set in Ableton (or any Live set)
  • Create a new MIDI track: Cmd+Shift+T
  • In the Browser sidebar, navigate to Max for Live → Max Instrument
  • Drag Max Instrument onto the MIDI track
Why MIDI? M4L instruments must live on MIDI tracks. Even though the template builder doesn't process audio, it needs to be an instrument type to access the Live Object Model (LOM) for creating tracks.

2 Open the Max editor

  • Look at the device in the track's device chain (bottom of the Ableton window)
  • Click the wrench icon in the top-right corner of the device — the Max patcher editor opens
  • You'll see some default objects — select all with Cmd+A and delete them
  • You now have a blank canvas
Edit vs. Run mode: The lock icon at the bottom-left (or Cmd+E) toggles between edit mode (where you place/connect objects) and run mode (where you click buttons). Make sure you're in edit mode (unlocked) for the next steps.

3 Add the JS object

  • Press N to create a new object box
  • Type: js stemforge_template_builder.js
  • Click outside the box to confirm
  • The object appears with 1 inlet (top) and 2 outlets (bottom)
[js stemforge_template_builder.js]
Max JavaScript object — the brain of this device
  • Inlet 0 (top): receives bang to start the build
  • Outlet 0 (bottom-left): status messages (text)
  • Outlet 1 (bottom-right): bang when build is complete

4 Place the JS file where Max can find it

  • Option A (recommended): Save the device first — File → Save As → save into your m4l/ folder. Then make sure stemforge_template_builder.js is in that same folder.
  • Option B: In the Max editor menu bar, go to Options → File Preferences and add the path to your m4l/ folder.
If the [js] object shows an error like "can't find file", it means Max can't locate the .js file. Double-check the filename spelling and that the file is in a search path.

5 Add the UI objects

You need three objects. For each one: press N, type the name, click outside to create it.

A) Button — triggers the build

[button]
Bang button — click to send a bang message
  • Press N, type button, click outside
  • Position it above the [js] object
  • It appears as a small circular button

B) Textedit — status log display

[textedit]
Multi-line text display — shows build progress
  • Press N, type textedit, click outside
  • Position it below-left of the [js] object
  • Resize it by dragging the bottom-right corner — make it wide enough to read status messages

C) Print — completion log (optional)

[print TemplateBuilder]
Prints to the Max console — useful for debugging
  • Press N, type print TemplateBuilder, click outside
  • Position it below-right of the [js] object
  • Output appears in the Max Console window (Window → Max Console)

6 Wire everything together

To connect two objects: hover over an outlet (bottom of source object) until the cursor changes, then click and drag to the inlet (top of destination object). A patch cord appears.

From Port To Port What it does
[button] outlet [js] inlet 0 Sends bang → calls bang() in the script
[js] outlet 0 (left) [textedit] inlet Status messages appear in the text box
[js] outlet 1 (right) [print] inlet Logs "bang" to Max Console on completion
Your finished patch: [button] | [js stemforge_template_builder.js] | | [textedit] [print TemplateBuilder] status log completion bang

7 Test the build

  • Switch to run mode: click the lock icon or press Cmd+E
  • Click the button — watch the textedit box fill with status messages
  • You should see messages like:
    Created track 1/7: SF | Drums Raw
    Created track 2/7: SF | Drums Crushed
    ...
    Template build complete! 7 tracks created.
  • Check Ableton — you should see 7 new tracks with the correct names and colors
If nothing happens: Make sure you're in run mode (locked). In edit mode, clicking the button selects it instead of triggering it.

8 Save and freeze

  • File → Save (or Save As → m4l/StemForgeTemplateBuilder.amxd)
  • File → Freeze Device — this bundles the .js file inside the .amxd so the device is portable

9 After the build completes

  • VST3 plugin parameters are left at defaults — dial them in per the Setup Guide
  • Select all 7 "SF |" tracks → Cmd+G to group → name "StemForge Templates" → color grey
  • Remove or bypass the builder device — it's a one-time tool

DEVICE 2 Stem Loader

Permanent device. Watches ~/stemforge/processed/ for new stems and loads them into the template tracks. Lives on its own MIDI track.

1 Create the Loader host track

  • In your StemForge Templates set, create a new MIDI track
  • Name it "SF Loader"
  • Drag Max Instrument onto it (same as before: Browser → Max for Live → Max Instrument)
  • Open the Max editor (wrench icon), delete all default objects

2 Add the core objects

This device has more objects than the builder. Create each one with N.

A) midiin — makes it a valid instrument

[midiin]
MIDI input — required for the device to sit on a MIDI track
  • Press N, type midiin, click outside
  • Place it in the top-left corner, out of the way — it doesn't connect to anything

B) js — the loader script

[js stemforge_loader.js]
Main logic — handles pipeline selection, folder watching, stem loading
  • Press N, type js stemforge_loader.js, click outside
  • Inlet 0: receives messages (loadPipeline, startWatch, stopWatch, bang)
  • Outlet 0: status text
  • Outlet 1: completion bang

C) loadbang + messages — initialization chain

[loadbang]
Fires a bang automatically when the device loads
  • Press N, type loadbang, click outside
  • Place it above the [js] object
[message setPipelinesDir ~/stemforge/pipelines]
Sets the pipelines directory on load
  • Press N, type message setPipelinesDir ~/stemforge/pipelines, click outside
  • The [message] box shows the text inside it
[message loadPipeline default]
Loads the default pipeline on startup
  • Press N, type message loadPipeline default, click outside

D) UI controls

[umenu]
Dropdown menu — lists available pipelines
  • Press N, type umenu, click outside
  • This populates with pipeline names from the pipelines directory
[toggle]
On/Off switch — starts/stops folder watching
  • Press N, type toggle, click outside
  • Label it "Watch" (Inspector → Long Name or add a [comment Watch] next to it)
[button]
Manual trigger — loads the latest stems on demand
  • Press N, type button, click outside
  • Label it "Load Latest"
[textedit]
Status display — shows what the loader is doing
  • Press N, type textedit, click outside
  • Resize it wide — this is the main status readout
[number]
Number display — shows BPM from the loaded manifest
  • Press N, type number, click outside
  • Label it "BPM"

3 Wire the Loader patch

From Port To Port Purpose
[loadbang] outlet [message setPipelinesDir ...] inlet Triggers the message on device load
[message setPipelinesDir ...] outlet [js] inlet 0 Tells the script where pipelines live
[loadbang] outlet [message loadPipeline default] inlet Auto-loads the default pipeline
[message loadPipeline default] outlet [js] inlet 0 Sends pipeline name to script
[umenu] outlet [message loadPipeline $1] inlet Dropdown selection → load command
[message loadPipeline $1] outlet [js] inlet 0 Sends selected pipeline to script
[toggle] outlet [message startWatch] / [message stopWatch] inlet Toggle → watch on/off (use [select 1 0] to route)
[button] outlet [js] inlet 0 Manual "Load Latest" trigger
[js] outlet 0 [textedit] inlet Status messages → display
[js] outlet 1 [print SFLoader] inlet Completion log to Max Console
Loader patch layout: [midiin] [loadbang] (unused) | \ [msg setPipelinesDir] [msg loadPipeline default] \ / [umenu] [toggle] [button] | | | [msg $1] [sel 1 0] | \ | / / [js stemforge_loader.js] | | [textedit] [print SFLoader] status log

4 Save, freeze, and install

  • File → Save As → m4l/StemForgeLoader.amxd
  • File → Freeze Device to bundle the JS file
  • Close the Max editor (click the wrench icon again)
  • The device stays on the "SF Loader" track permanently

REF Troubleshooting

"can't find file" on the [js] object
The .js file must be in the same folder as the .amxd, or in a Max search path (Options → File Preferences). Filenames are case-sensitive.
Nothing happens when I click the button
You're probably in edit mode. Press Cmd+E to switch to run mode (locked). In edit mode, clicks select objects instead of triggering them.
"NOT FOUND: PluginName" in the status log
The VST3 plugin isn't installed, or its Browser name doesn't match what the script searches for. Edit the search field in the TEMPLATES array in the JS file to match what you see in Ableton's Browser under Plug-ins.
Param not found
Ableton device parameter names vary between versions. Use the M4L Inspect workflow: select the device, read parameter names in the Max console, then update the params object in the script.
Max Console
If something isn't working, open the Max Console (Window → Max Console in the Max editor) to see detailed error messages and post() output from the JS objects.
Helpful shortcuts in the Max editor
N — new object  •  M — new message box  •  C — new comment  •  Cmd+E — toggle edit/run mode  •  Cmd+I — open Inspector for selected object
