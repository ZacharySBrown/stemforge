StemForge Loader — M4L Build Guide

Build the Stem Loader device in the Max for Live editor.
This device watches for new stems and sets up your Ableton session automatically.

Template Builder Guide Setup Guide

INFO What the Loader does

When triggered (manually or via watch mode), the Loader reads a stems.json manifest, then:

  • Sets the session BPM from the manifest
  • Duplicates your template tracks for each stem
  • Renames and recolors the new tracks
  • Sets device parameters from your pipeline config
  • Tells you which audio files to drag onto which tracks
ActionAutomated?Notes
Set BPM Auto Reads bpm from stems.json
Duplicate template tracks Auto Finds templates by name, duplicates via LOM
Rename + recolor tracks Auto e.g. "MyTrack | drums", "MyTrack | bass"
Set device parameters Auto From pipeline JSON config
Load audio onto tracks Manual LOM cannot load audio files — drag from Browser
Load sample into Simpler Manual Drag beat slice into Simpler's sample slot

1 Create the host track

  • In your StemForge Templates set, create a new MIDI track: Cmd+Shift+T
  • Name it "SF Loader"
  • From the Browser: Max for Live → Max Instrument — drag onto the track
  • Click the wrench icon to open the Max editor
  • Select all default objects (Cmd+A) and delete them

2 Save the device first

  • File → Save As → navigate to your m4l/ folder
  • Save as: StemForgeLoader.amxd
  • This ensures the [js] object can find stemforge_loader.js in the same folder
Do this before adding the [js] object. If Max can't find the JS file, the object will only show 1 outlet instead of 2.

3 Add all the objects

Make sure you're in edit mode (unlocked — Cmd+E). Press N to create each object, type the text shown, click outside to confirm. Press M to create message boxes (the ones starting with message below).

The brain

[js stemforge_loader.js]
Press N, type: js stemforge_loader.js
  • Inlet 0 (top): receives messages — bang, setProcessedDir, setPipelinesDir, startWatch, stopWatch, loadPipeline
  • Outlet 0 (bottom-left): status text for display
  • Outlet 1 (bottom-right): bang on load complete

Should show 2 outlets. If you only see 1, Max can't find the JS file — check that the .amxd and .js are in the same folder.

Initialization chain — runs on device load

[loadbang]
Press N, type: loadbang
  • Fires a bang automatically when the device is loaded in Ableton
  • Place near the top of the patch
[setProcessedDir ~/stemforge/processed]
Press M, type: setProcessedDir ~/stemforge/processed
  • Tells the script where to look for new stems
  • Edit the path if your stemforge folder is elsewhere
[setPipelinesDir ~/stemforge/pipelines]
Press M, type: setPipelinesDir ~/stemforge/pipelines
  • Tells the script where pipeline configs live
[loadPipeline default]
Press M, type: loadPipeline default
  • Auto-loads the default pipeline config on startup
Message boxes vs. object boxes: Press M for message boxes (they have a flag/tab on the right side). Press N for object boxes (plain rectangle). The three items above are message boxes.

User controls

[button]
Press N, type: button
  • The "Load Latest" button — click to manually trigger stem loading
  • Add a [comment Load Latest] next to it as a label (press C, type the label)
[toggle]
Press N, type: toggle
  • Watch mode on/off switch
  • Add a [comment Watch] label next to it
[select 1 0]
Press N, type: select 1 0
  • Routes the toggle output: 1 (on) goes to outlet 0, 0 (off) goes to outlet 1
[startWatch]
Press M, type: startWatch
  • Sent when toggle turns ON
[stopWatch]
Press M, type: stopWatch
  • Sent when toggle turns OFF

Output display

[textedit]
Press N, type: textedit
  • Main status display — shows loading progress and drag instructions
  • Resize it wide and tall (drag the bottom-right corner) so you can read the output
[print SFLoader]
Press N, type: print SFLoader
  • Logs completion to the Max Console

4 Arrange the objects

Lay them out roughly like this — signals flow top to bottom. Keep the [js] object in the center since most things connect to it.

Suggested layout: [loadbang] / | \ [msg setProcessedDir] [msg setPipelinesDir] [msg loadPipeline default] \ | / \ | / [button] \ | / [toggle] | \ | / | | \ | / [select 1 0] | \ | / | | | \ | / [msg startWatch] [msg stopWatch] | \| / \ / | | \ / +-----------+---------------+---+ | [js stemforge_loader.js] | | [textedit] [print SFLoader] status display completion log

5 Wire everything

To connect: hover over an outlet (bottom of object) until the cursor changes, then click and drag to the inlet (top of target object).

How to count inlets and outlets: Inlets are the small tabs on the top of an object, outlets are on the bottom. They're numbered left to right, starting at 0. So "inlet 0" is always the leftmost tab on top. If an object only has one inlet, it's inlet 0. Hover over any inlet/outlet — Max shows a tooltip with the number.

Initialization wiring

FromPortToPortPurpose
[loadbang] outlet 0 (only outlet) [msg setProcessedDir ...] inlet 0 (only inlet) Triggers processed dir path on load
[loadbang] outlet 0 (same outlet) [msg setPipelinesDir ...] inlet 0 (only inlet) Triggers pipelines dir path on load
[loadbang] outlet 0 (same outlet) [msg loadPipeline default] inlet 0 (only inlet) Loads default pipeline on startup
[msg setProcessedDir ...] outlet 0 (only outlet) [js] inlet 0 (only inlet) Sends path to script
[msg setPipelinesDir ...] outlet 0 (only outlet) [js] inlet 0 (only inlet) Sends path to script
[msg loadPipeline default] outlet 0 (only outlet) [js] inlet 0 (only inlet) Loads pipeline config
One outlet, many destinations: [loadbang] connects to all three message boxes. Just drag three separate patch cords from its outlet. They all fire simultaneously on load.

User control wiring

FromPortToPortPurpose
[button] outlet 0 (only outlet) [js] inlet 0 (only inlet) Manual "Load Latest" — sends bang
[toggle] outlet 0 (only outlet) [select 1 0] inlet 0 (leftmost — has 3 inlets, use the left one) Routes on/off to separate messages
[select 1 0] outlet 0 (leftmost of 3) [msg startWatch] inlet 0 (only inlet) Toggle ON → fires when input matches 1
[select 1 0] outlet 1 (middle of 3) [msg stopWatch] inlet 0 (only inlet) Toggle OFF → fires when input matches 0
[msg startWatch] outlet 0 (only outlet) [js] inlet 0 (only inlet) Sends startWatch to script
[msg stopWatch] outlet 0 (only outlet) [js] inlet 0 (only inlet) Sends stopWatch to script
About [select 1 0]: This object has 3 inlets and 3 outlets. Always connect to inlet 0 (leftmost). The outlets: outlet 0 (left) fires when the input matches 1, outlet 1 (middle) fires when input matches 0, outlet 2 (right) fires for anything else — you can ignore outlet 2.

Output wiring

FromPortToPortPurpose
[js] outlet 0 (leftmost of 2) [textedit] inlet 0 (only inlet) Status messages appear in the text box
[js] outlet 1 (rightmost of 2) [print SFLoader] inlet 0 (only inlet) Logs "bang" to Max Console on completion

6 Test it

  • Switch to run mode: Cmd+E (lock icon should be locked)
  • You should see initialization messages in the textedit:
    Processed dir: ~/stemforge/processed
    Pipelines dir: ~/stemforge/pipelines
  • Click the button to try loading — if no stems have been processed yet, you'll see: No stems.json found in ~/stemforge/processed
  • Turn on the toggle — should show: Watching ~/stemforge/processed
  • Turn it off — should show: Watch stopped
Nothing in textedit?
1. Make sure you're in run mode (locked).
2. Check the Max Console (Window → Max Console) — messages always appear there via post().
3. Verify the patch cord goes from [js] left outlet (outlet 0) to [textedit].

7 Save and freeze

  • File → Save
  • File → Freeze Device — bundles stemforge_loader.js inside the .amxd
  • Close the Max editor (click the wrench icon again)
  • The device stays on the "SF Loader" track permanently

USE Day-to-day workflow

  • Run stemforge from the CLI to process a track — stems land in ~/stemforge/processed/
  • If watch mode is on, the Loader detects the new stems.json automatically
  • Otherwise, click Load Latest
  • The Loader duplicates your template tracks, renames them, sets BPM and device params
  • Check the textedit output — it tells you exactly which stems to drag onto which tracks
  • In Ableton's Browser, navigate to the processed/ folder and drag the stems onto the indicated tracks

REF Troubleshooting

"can't find file stemforge_loader.js"
The .js and .amxd must be in the same folder. Save the device into m4l/ where the JS file lives, then delete and re-create the [js] object.
Only 1 outlet on the [js] object
Same cause — Max can't find the JS file, so it doesn't read outlets = 2. Fix the file path, then re-create the object.
textedit shows "doesn't understand"
The [js] outlet must send set <text> to textedit. This is handled in the script's status() function. If you see this error, make sure you're using the latest version of stemforge_loader.js.
"No stems.json found"
Run stemforge from the CLI first to process a track. The Loader looks for stems.json files inside subdirectories of ~/stemforge/processed/.
"Template not found: SF | Drums Raw"
The Loader finds template tracks by exact name. Make sure your template tracks are named exactly as shown in the Setup Guide (e.g. SF | Drums Raw, not SF| Drums Raw).
