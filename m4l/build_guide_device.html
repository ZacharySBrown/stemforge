StemForge Integrated Forge Device

One-button split → slice → curate → play pipeline, driven from an Ableton audio clip through a Max for Live device.

1 Install the Python backend

  • From a terminal in the repo root: ./install.sh
  • When it finishes, verify ~/.stemforge/python_path exists and points at a python executable (usually <repo>/.venv/bin/python).
  • The device uses this file to find the interpreter; Max does not inherit your shell PATH.
Re-run any time your venv moves. The install script is idempotent and rewrites python_path on each run.

2 Open the patch in Max

  • Open Max 8 or Max 9 (standalone, not from within Ableton the first time).
  • File → Open → select m4l/StemForgeDevice.maxpat from the repo.
  • The patch should load with a FORGE button, strategy menu, n_bars numbox, 4×4 pad grid, four stem dials, and a spectroscope.
  • Make sure stemforge_bridge.js, stemforge_lom.js, and package.json are in the same folder as the .maxpat.

3 Save as an Audio Effect

  • File → Save As → navigate to your Ableton User Library: ~/Music/Ableton/User Library/Presets/Audio Effects/Max Audio Effect/StemForge/ (create the folder if needed).
  • Save as StemForgeDevice.amxd. Set the device type to Audio Effect.
  • After saving, choose File → Freeze Device to bundle the JS files into the .amxd.
Why Audio Effect? The forge device reads audio clips from the track it sits on, so it must live on an audio track alongside the source clip.

4 Drop the device on a track

  • In Ableton, drop an audio clip onto an audio track. Let Ableton auto-warp it (or correct warp markers manually if the groove is complex).
  • From the Browser → Max for Live → Max Audio Effect, drag StemForgeDevice onto the same track.
  • Select the clip slot holding your audio so the device knows which clip to analyze.

5 Press FORGE

  • Pick a Strategy (max-diversity is a good default) and choose n_bars (12–16).
  • Click FORGE. The status line will show started → splitting → slicing → curating → done.
  • When complete, the pad grid unlocks and each pad plays one curated bar. Pads are time-stretched via groove~ so they stay locked to session tempo.
  • Click CANCEL at any time to kill the Python subprocess.

6 MIDI-map the pads

  • Press Cmd+M in Ableton to enter MIDI map mode.
  • Click a cell in the 4×4 pad grid and play a pad/key on your controller (Push, Launchpad, keyboard). Repeat for each pad.
  • Exit MIDI map mode with Cmd+M. Pads will now trigger from hardware.

7 Troubleshooting

"Python backend not found"

  • The device could not read ~/.stemforge/python_path. Re-run ./install.sh.
  • If you moved the repo or the venv, rewrite the file manually: echo /abs/path/to/.venv/bin/python > ~/.stemforge/python_path.

First run hangs on "splitting"

  • Demucs downloads an ~80 MB model to ~/.cache/torch/hub/ on first use. The status line shows downloading model... when it detects this.
  • On slow connections this can take a few minutes. Subsequent runs skip the download.

Odd time signatures (7/4, 6/8, 5/4)

  • Set the clip's time signature correctly in Ableton before forging. The device reads signature_numerator / signature_denominator from the clip.
  • Verify warp markers are present and correct. Without markers, the device falls back to a rigid grid and flags analysis_fallback: true in the status.

Pads do not trigger after FORGE completes

  • Check the status line for errors. The pad gate only opens after a ready N event from the bridge.
  • If curated/drums/ is empty, the backend selected no bars — lower the RMS floor in the CLI or try a different strategy.

"curated stem dir not found"

  • The default focus stem is drums. If your Demucs backend produced different stem names, send setFocusStem <name> to the node.script object (e.g. bass, vocals, other).
